4 Common JS Questions and Answers

1. Difference between local storage and session storage:

Local Storage:

  1. Local storage is a more modern storage space for web pages. Its size capacity could reach up to 5MB/10MB, depending on browser.
  2. Information stored in local storage will not be deleted. One must delete them manually or wait for JavaScript to call localStorage.clear() or localStorage.removeItem(key).
  3. However, data stored in local storage will not be sent to server automatically with HTTP request.
localStorage.setItem('myCat', 'Tom'); // setItem

const cat = localStorage.getItem('myCat'); // getItem

localStorage.removeItem('myCat'); // removeItem

localStorage.clear(); // clear all

Session Storage:

  1. The storage capacity of session storage is 5MB
  2. Unlike local storage, information in session storage will be deleted when the session ends, for example, closing browser.
  3. Data is only available during the page session. Changes made are saved and available for the current page, as well as future visits to the site on the same tab/window. Once the tab/window is closed, the data is deleted.

2. Difference between global scope and block scope in JavaScript:

The Global scope:

A variable exists inside or outside a block. If a variable is declared outside all functions or curly braces ({}), it exists in the global scope. The global variables can be accessed by any line of code in the program, including inside blocks.

// program to print a text
let a = "hello";
function greet () {
console.log(a);
}
greet(); // hello

The Block Scope:

A block scoped variable means that the variable defined within a block will not be accessible from outside the block. A block can reside inside a function, and a block scoped variable will not be available outside the block even if the block is inside a function.

3. What is an event loop in JavaScript?

The event loop:

  1. The event loop is the secret behind JavaScript’s asynchronous programming. JS executes all operations on a single thread, but using a few smart data structures, it gives us the illusion of multi-threading. Let’s take a look at what happens on the back-end.
  2. The call stack is responsible for keeping track of all the operations in line to be executed. Whenever a function is finished, it is popped from the stack.
  3. The event queue is responsible for sending new functions to the stack for processing. It follows the queue data structure to maintain the correct sequence in which all operations should be sent for execution.

Function calls form a stack of frames.

function foo(b) {
const a = 10;
return a + b + 11;
}

function bar(x) {
const y = 3;
return foo(x * y);
}

const baz = bar(7); // assigns 42 to baz

Event loop

The event loop got its name because of how it's usually implemented, which usually resembles:

while (queue.waitForMessage()) {
queue.processNextMessage();
}

queue.waitForMessage() waits synchronously for a message to arrive (if one is not already available and waiting to be handled).

4. The ways we can get undefined:

undefined value:

you will receive undefined when accessing uninitialized variables, non-existing object properties, non-existing array elements, and alike.

Scenarios that create undefined:

1. Uninitialized variable:

A declared variable but not yet assigned with a value (uninitialized) is by default undefined.

let myVariable;
myVariable; // => undefined

2. Accessing a non-existing property:

When accessing a non-existing object property, JavaScript returns undefined.

let favoriteMovie = {
title: 'Blade Runner'
};
favoriteMovie.actors; // => undefined

3. Function parameters:

The function parameters implicitly default to undefined.

function multiply(a, b) { //a => 5, b => undefined
return a * b;
}
multiply(5); // => NaN

4. Function return value:

Implicitly, without return statement, a JavaScript function returns undefined.

function square(x) {
const res = x * x;
}
square(2); // => undefined

5. void operator:

void expression evaluates the expression and returns undefined no matter the result of the evaluation.

void 1; // => undefined
void (false); // => undefined
void {name: 'John Smith'}; // => undefined
void Math.min(1, 3); // => undefined

6. undefined in arrays:

You get undefined when accessing an array element with an out of bounds index.

const colors = ['blue', 'white', 'red'];
colors[5]; // => undefined
colors[-1]; // => undefined
